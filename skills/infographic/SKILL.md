# インフォグラフィック生成スキル

Gemini API を使ってインフォグラフィック画像を生成する。

## スクリプトの場所

スクリプトはこのSKILL.mdと同じプラグイン内にある。
パス参照には `${CLAUDE_PLUGIN_ROOT}` 環境変数を使う（プラグインシステムが自動で設定する）。

```
${CLAUDE_PLUGIN_ROOT}/
├── skills/infographic/SKILL.md  ← このファイル
├── scripts/
│   ├── generate-image.py
│   ├── gemini_client.py
│   ├── prompt_builder.py
│   ├── presets.json
│   └── styles/
│       ├── consulting.txt
│       └── graphic-recording.txt
└── .env (任意)
```

- デフォルトモデル: `gemini-3-pro-image-preview`
- API設定: 環境変数 `Google_Image_API` またはユーザーのプロジェクトルートの `.env`

## 引数

```
$ARGUMENTS = "{構成タイプ} {トピックや指示}"
```

- 構成タイプが省略された場合は用途から判断する
- トピックが省略された場合は対話で確認する

## 実行フロー

### Step 1: プリセットを選択する

まず `scripts/presets.json` を読み込み、ユーザーに構成とスタイルを確認する。

#### 構成（Composition）

| # | 構成 | 説明 | デフォルト比率 |
|---|------|------|---------------|
| 1 | **プレゼン用スライド** | キーメッセージ1つ + ビジュアル。文字は最小限 | 16:9 |
| 2 | **ビジネス資料** | 情報量多め。フレームワーク図・比較表・フロー図 | 16:9 |
| 3 | **ドキュメント用インフォグラフィック** | 記事・ブログ・社内ドキュメントに埋め込む図解 | 4:3 |
| 4 | **モチーフ／イラスト素材** | アイコン・イラスト単体。テキストなし | 1:1 |
| 5 | **カスタム** | プリセットなし、自由にプロンプト設計 | — |

#### スタイル（Tone）

| スタイル | 特徴 | ファイル |
|---------|------|---------|
| **IDEO / McKinsey コンサル風** | オフホワイト背景、フラットデザイン、手書き風矢印 | `styles/consulting.txt` |
| **グラフィックレコーディング風** | 手描きマーカー、吹き出し、ストーリーマップ的配置 | `styles/graphic-recording.txt` |
| **カスタム** | ユーザーがスタイル指示を直接指定 | — |

**確認例:**
> 「プレゼン用スライドをコンサル風で作りますか？それとも別の構成・スタイルがいいですか？」

ユーザーの回答から以下を確定する：
- **構成**: 上記1-5のいずれか
- **スタイル**: 上記のいずれか or カスタム
- **アスペクト比**: プリセットのデフォルトを提案（ユーザーが変更可）
- **テーマ・内容**: 何の図解/素材か
- **枚数**: 1枚 or 複数枚
- **出力先**: デフォルトは `output/{用途フォルダ}/`（現在のプロジェクトルート相対）。ユーザーが別のパスを指定した場合はそちらを使う

**出力先の確認例:**
> 「出力先は `output/presentation/` でよいですか？別のフォルダに保存したい場合は指定してください。」

### Step 2: プロンプトを設計する

**基本原則: 日本語でプロンプトを書く。** Gemini は日本語テキストの描画精度が高い。

選択されたプリセットに基づいてプロンプトを組み立てる：

```
[スタイル] ← styles/{name}.txt から読み込み、--style-file で指定
[コンテンツ] ← ユーザーの指示 + プリセットの prompt_suffix をガイドに
[制約] ← アスペクト比は --aspect で指定
```

#### 構成別のプロンプト設計ガイド

**プレゼン用スライド:**
- キーメッセージを大きく配置する
- 補足テキストは最小限（3-5語程度のラベル）
- ビジュアル要素で伝える。テキストの壁にしない
- 要素の配置を「左側」「右側」「中央」など空間的に指示

**ビジネス資料:**
- 情報を構造化して配置する（2x2マトリクス、フロー図、比較表等）
- テキスト内容を正確に書く（モデルがそのまま描画する）
- 見出し + 本文 + 図 の階層を意識する
- 色でカテゴリを分ける

**ドキュメント用インフォグラフィック:**
- 記事の要点を視覚的にまとめる
- 数字やキーワードを目立たせる
- 読者がスクロール中に目を止めるレイアウト

**モチーフ／イラスト素材:**
- **必ず「テキストや文字は含めない」を明示する** — モデルは指示がないとテキストを入れがち
- 背景はシンプル（白 or 単色）
- 1モチーフ1プロンプト（複数要素を詰め込まない）
- 「フラットデザイン」「線画」「ピクトグラム」等のスタイルを明示

**モチーフプロンプトの例:**
```
フラットデザインのアイコンイラスト。
走り回っている人の全身像。オレンジ色をメインカラーに。
忙しそうに複数のタスクをこなしている様子。
背景は白。テキストや文字は含めない。
シンプルで洗練されたビジネス向けのスタイル。
```

**プロンプトをユーザーに提示して確認を取る。**

### Step 3: 画像を生成する

`${CLAUDE_PLUGIN_ROOT}` でスクリプトとスタイルファイルを参照する。出力先はユーザーのプロジェクト内（cwdからの相対パス）。

```bash
# プレゼン用スライド（コンサル風、16:9）
python3 "${CLAUDE_PLUGIN_ROOT}/scripts/generate-image.py" \
  --prompt "日本語プロンプト" \
  --style-file "${CLAUDE_PLUGIN_ROOT}/scripts/styles/consulting.txt" \
  --aspect 16:9 \
  --output output/{用途フォルダ}/{ファイル名}.png

# ビジネス資料（グラレコ風、16:9）
python3 "${CLAUDE_PLUGIN_ROOT}/scripts/generate-image.py" \
  --prompt "日本語プロンプト" \
  --style-file "${CLAUDE_PLUGIN_ROOT}/scripts/styles/graphic-recording.txt" \
  --aspect 16:9 \
  --output output/{用途フォルダ}/{ファイル名}.png

# ドキュメント用（コンサル風、4:3）
python3 "${CLAUDE_PLUGIN_ROOT}/scripts/generate-image.py" \
  --prompt "日本語プロンプト" \
  --style-file "${CLAUDE_PLUGIN_ROOT}/scripts/styles/consulting.txt" \
  --aspect 4:3 \
  --output output/{用途フォルダ}/{ファイル名}.png

# モチーフ（スタイルなし、1:1）
python3 "${CLAUDE_PLUGIN_ROOT}/scripts/generate-image.py" \
  --prompt "日本語プロンプト" \
  --aspect 1:1 \
  --output output/{用途フォルダ}/motif-{名前}.png

# カスタム（スタイルテキスト直接指定）
python3 "${CLAUDE_PLUGIN_ROOT}/scripts/generate-image.py" \
  --prompt "日本語プロンプト" \
  --style-prefix "Style: dark tech..." \
  --aspect 16:9 \
  --output output/{用途フォルダ}/{ファイル名}.png

# プロンプトファイルから（長文の場合）
python3 "${CLAUDE_PLUGIN_ROOT}/scripts/generate-image.py" \
  --prompt-file output/{用途フォルダ}/prompt-01.txt \
  --style-file "${CLAUDE_PLUGIN_ROOT}/scripts/styles/consulting.txt" \
  --output output/{用途フォルダ}/slide-01.png

# 複数枚（バリエーション）
python3 "${CLAUDE_PLUGIN_ROOT}/scripts/generate-image.py" \
  --prompt "..." \
  --num 3 \
  --output-dir output/{用途フォルダ}/ \
  --name motif-runner

# 低コスト版を使いたい場合
python3 "${CLAUDE_PLUGIN_ROOT}/scripts/generate-image.py" \
  --model gemini-3.1-flash-image-preview \
  --prompt "..." \
  --output output/{用途フォルダ}/{ファイル名}.png
```

### Step 4: セルフレビュー

生成された画像を `Read` ツールで表示し、構成タイプに応じてレビューする：

**テキスト系（プレゼン・ビジネス資料・ドキュメント）:**
1. **日本語テキストの正確性** — 文字化け・脱字・重複がないか
2. **内容の一致** — プロンプトで指示した要素が全て含まれているか
3. **レイアウト** — 要素の配置がプロンプトの意図通りか
4. **可読性** — テキストが小さすぎないか、背景と被っていないか
5. **トンマナ** — 選択したスタイルが反映されているか

**モチーフ:**
1. **不要なテキストが入っていないか** — 入っていたらプロンプトを強化して再生成
2. **モチーフの明確さ** — 何を表しているかが一目でわかるか
3. **背景** — 白背景 or 単色になっているか
4. **色味** — 指定したアクセントカラーが反映されているか
5. **素材としての使いやすさ** — 他のスライドやツールに貼り込みやすい構図か

**レビュー結果に基づくアクション:**
- 全てOK → Step 5へ
- 軽微な問題（配色、レイアウトの微調整） → プロンプトを修正して再生成
- 重大な問題（テキスト崩壊、要素欠落） → プロンプトを大幅に見直して再生成

### Step 4b: リトライ制限

**同一画像のリトライは最大5回まで。** 5回リトライしても満足いく結果が出ない場合は：
1. これまでの生成結果をユーザーに見せる
2. 問題点を報告する
3. 以下の選択肢を提示する：
   - プロンプトの方向性を変える
   - テキスト量を減らす / 英語に切り替える
   - スタイルを変える（コンサル風 → グラレコ風 等）
   - この画像はスキップする
   - 別のモデルを試す

**絶対にループしてコストを無駄にしない。**

### Step 5: ユーザー確認

- 生成された画像をユーザーに見せる
- フィードバックがあれば反映（Step 4bのリトライ上限を守る）
- OKが出たら次の画像に進む

### Step 6: 出力物を整理する

- 最終版のファイル名を整理する（不要な中間ファイルは削除提案）
- プロンプトも保存しておく（再生成用）: `output/{用途フォルダ}/prompts.md`

## モデル選択ガイド

| モデル | ID | 品質 | 用途 |
|--------|-----|------|------|
| **Gemini 3 Pro** | `gemini-3-pro-image-preview` | 最高 | デフォルト。正式な資料・記事用 |
| **Gemini 3.1 Flash** | `gemini-3.1-flash-image-preview` | 高 | ドラフト・大量生成・コスト重視 |
| **Imagen 4** | `imagen-4.0-generate-001` | 写実的 | 写真風のビジュアルが必要な場合 |

## スタイルの追加方法

1. `scripts/styles/` に新しい `.txt` ファイルを作成する
2. `scripts/presets.json` の `styles` に追加する
3. SKILL.md のスタイルテーブルに追記する

**スタイルファイルのフォーマット:**
```
Style: [全体のトーン・雰囲気の説明]
[背景の指示]
[タイポグラフィの指示]
[配色の指示]
[レイアウト・装飾の指示]
[品質の指示]
```

## トラブルシューティング

| 症状 | 対処 |
|------|------|
| API key not found | `.env` に `Google_Image_API` が設定されているか確認 |
| HTTP 429 | レート制限。少し待ってリトライ |
| HTTP 400 | プロンプトが不適切と判断された可能性。内容を調整 |
| HTTP 404 | モデルIDが間違っている。`--model` を確認 |
| 日本語テキストが崩れる | Gemini 3 Pro を使う。それでも崩れる場合はテキスト量を減らす |
| Content blocked | プロンプトの表現を変更する |
